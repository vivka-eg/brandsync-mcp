# Personal Details

**Type:** UI Pattern (full screen)
**Category:** detail, profile
**Responsive:** Desktop, Tablet, Mobile

## Use Case

Display and edit detailed information about a single record — a person, customer, asset, or entity. Use when a user navigates from a list (User Management) into an individual record. Supports view and edit modes.

## Components Used

- Input Fields
- Buttons
- Avatar
- Tabs
- Card
- Badge
- List
- Breadcrumb
- Navigation Drawer
- Menu
- Switch

## Design Tokens

- `--bs-color-surface-base`
- `--bs-color-surface-raised`
- `--bs-color-text-default`
- `--bs-color-text-secondary`
- `--bs-color-text-muted`
- `--bs-color-border-default`
- `--bs-color-primary-default`
- `--bs-color-text-inverse`

## Layout

App shell with sidebar, breadcrumb, profile header with avatar, tabbed content sections (Overview, Documents, History, Notes).

## States

- View Mode
- Edit Mode
- Saving

## Dark Mode

Supported via `data-theme="dark"`
