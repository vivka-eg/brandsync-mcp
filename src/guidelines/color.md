# Color Guidelines

## Brand Colors

Brandsync uses a semantic color system built on CSS custom properties with the `--bs-color-*` prefix.

### Primary Palette

- `--bs-color-primary-default` — main brand color, used for CTAs and key interactive elements.
- `--bs-color-primary-hover` — slightly darker shade for hover states.
- `--bs-color-primary-active` — pressed/active state.
- `--bs-color-primary-subtle` — light tint for backgrounds and badges.

### Neutral Palette

- `--bs-color-neutral-0` — white
- `--bs-color-neutral-100` through `--bs-color-neutral-900` — gray scale
- `--bs-color-neutral-1000` — black

### Semantic Colors

| Token | Usage |
|---|---|
| `--bs-color-success-default` | Positive feedback, completed states |
| `--bs-color-warning-default` | Caution, destructive confirmations |
| `--bs-color-danger-default` | Errors, destructive actions |
| `--bs-color-info-default` | Informational messages |

## Accessibility

All color combinations must meet WCAG 2.1 AA contrast ratio (4.5:1 for normal text, 3:1 for large text and UI components).

Never convey information using color alone — pair color with an icon or text label.
